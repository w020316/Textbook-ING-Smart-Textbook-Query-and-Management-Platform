### 1.3 根因分析

#### 问题1根因：前后台登录状态共享

**原设计缺陷**：前台 `/login` 和后台 `/admin/login` 共用同一个 localStorage key（`token` + `user`）。

**问题链条**：
1. 管理员通过 `/admin/login` 登录 → token 存入 `localStorage.token`
2. 管理员访问前台点击"登录" → 路由跳转 `/login`
3. 路由守卫检查 `localStorage.token` 存在 → 判断"已登录" → 强制跳转首页
4. 用户看到 `/login` URL 但显示首页内容

**代码位置**：
- `src/router/index.ts` 第213-216行（修复前）：
```typescript
// 已登录访问前台登录/注册页 → 跳转首页
if (isAuthed && (to.name === 'login' || to.name === 'register')) {
  next({ path: '/' })
  return
}
```

#### 问题7/8根因：公开接口误配置认证

**原配置错误**：`api/index.ts` 路由表中，`/api/colleges` 和 `/api/textbooks/hot-searches` 等公开接口被错误标记为 `auth: true`。

```typescript
// 修复前（错误）
{ method: 'GET', pattern: '/api/colleges', handler: handleColleges, auth: true },
{ method: 'GET', pattern: '/api/textbooks/hot-searches', handler: handleHotSearches, auth: true },
```

#### 问题3根因：种子数据多学期重复创建教材

**原 seed.ts 逻辑**：每个课程在最近3个学期各创建一个实例，每个实例都创建一份教材，导致同一教材出现3份记录。

```typescript
// 修复前（产生重复）
for (const c of courses) {
  for (const sem of semesterRecords.slice(-3)) {
    const course = await prisma.course.create({...})
    for (const tb of c.textbooks) {
      await prisma.textbook.create({...})  // 每学期都创建
    }
  }
}
```

### 2.2 详细修复方案

#### 修复1：前后台登录状态隔离（问题1）

**方案**：管理后台使用独立的 localStorage key（`adminToken` + `adminUser`），与前台完全隔离。

**新增文件**：
- `src/stores/adminAuth.ts` - 管理后台独立认证 store

**修改文件**：
- `src/utils/request.ts` - 支持 adminToken，按 URL 前缀自动选择 token
  - `/admin` 开头请求 → 使用 `adminToken`
  - 其他请求 → 使用 `token`
- `src/router/index.ts` - 路由守卫分别判断前后台登录态
- `src/admin/views/AdminLoginView.vue` - 改用 adminAuthStore
- `src/admin/layouts/AdminLayout.vue` - 退出登录改用 adminAuthStore
- `src/admin/views/DashboardView.vue` - 用户信息改用 adminAuthStore
- `src/views/LoginView.vue` - 拒绝管理员账号前台登录

**核心代码**：
```typescript
// src/utils/request.ts - 按 URL 前缀选择 token
function pickToken(url: string): string | null {
  const normalized = url.startsWith('/') ? url : `/${url}`
  if (normalized.startsWith('/admin')) {
    return getAdminToken()
  }
  return getToken()
}

// src/router/index.ts - 分别判断前后台登录态
const frontToken = localStorage.getItem('token')
const isAdminAuthed = !!localStorage.getItem('adminToken')
// 前台路由只看 frontToken，后台路由只看 adminToken
```

#### 修复2：公开API缓存优化（问题2/6）

**方案**：为低频变更的公开接口添加 LRU+TTL 缓存，管理后台写操作后主动失效。

**缓存策略**：

| 接口 | TTL | 理由 |
|------|-----|------|
| /api/stats | 5 分钟 | 首页统计，平衡实时性 |
| /api/semesters | 30 分钟 | 学期低频变更 |
| /api/calendar | 30 分钟 | 校历低频变更 |
| /api/news/categories | 60 分钟 | 分类极少变更 |
| /api/news（首页） | 2 分钟 | 新闻列表首页短期缓存 |
| /api/colleges | 60 分钟 | 学院低频变更 |
| /api/textbooks/hot-searches | 10 分钟 | 热门搜索 |

**缓存失效**：管理后台增删改操作后调用 `invalidateCache(prefix)` 主动失效相关缓存。

#### 修复3：教材数据去重（问题3）

**方案**：
1. 修复 `prisma/seed.ts`：教材只关联到课程的第一个学期实例
2. 新增 `scripts/dedup-textbooks.cjs` 去重脚本清理现有数据
3. 执行结果：45 本教材 → 15 本（删除 30 本重复）

#### 修复4：凭据环境变量化（问题4）

**方案**：4个测试脚本中的硬编码凭据改为读取环境变量 `TEST_ADMIN_EMAIL` / `TEST_ADMIN_PASSWORD`，提供开发默认值。

**涉及脚本**：
- `scripts/test-full-api.cjs`
- `scripts/final-verify.cjs`
- `scripts/verify-deploy.cjs`
- `scripts/verify-deploy-v2.cjs`

#### 修复5：登录后返回原页面（问题5）

**方案**：`SearchView.vue` 的"立即登录"按钮跳转时携带 `redirect` 参数。

```typescript
function goLogin() {
  router.push({ path: '/login', query: { redirect: route.fullPath } })
}
```

#### 修复7/8：公开接口认证修复

**方案**：移除 `/api/colleges`、`/api/colleges/:id/majors`、`/api/majors/:id/classes`、`/api/textbooks/hot-searches` 的 `auth: true` 标记。
