## 1. 任务编号规则与说明

### 1.1 编号规则

- 任务 ID 格式：`<分类>-<序号>`，如 `FE-01`、`BE-05`、`DB-03`。
- 分类前缀：
  - `FE` - Frontend 前端
  - `BE` - Backend 后端
  - `DB` - Database 数据库
  - `QA` - Quality Assurance 测试
  - `OPS` - DevOps 部署

### 1.2 复杂度说明

复杂度采用 1-5 分制，对应预估工时：

| 复杂度 | 说明 | 预估工时 |
| --- | --- | --- |
| 1 | 极简，配置或基础搭建 | 0.5 人天 |
| 2 | 简单，单组件 / 单接口 CRUD | 1 人天 |
| 3 | 中等，含业务逻辑 / 联动 / 状态管理 | 2 人天 |
| 4 | 较复杂，跨模块联动 / 异步流程 | 3 人天 |
| 5 | 复杂，核心流程 / 性能优化 / 安全相关 | 5 人天 |

### 1.3 依赖关系标注

- `依赖：xxx` 表示该任务需在 xxx 完成后开始。
- `无` 表示无前置依赖，可并行。
- `可并行` 表示可与同批次任务并行。
