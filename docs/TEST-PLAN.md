## 1. 测试策略

### 1.1 测试目标

- 验证教材ING 平台重建版本 V12.0.0 满足 PRD 中所有 P0/P1 功能需求。
- 确保覆盖线上 V11.3.0 的全部功能（注册/登录/忘记密码/教材查询/校历/新闻/评论/积分/消息/热门搜索）。
- 保证系统在性能、安全、兼容性方面达到非功能性需求指标。
- 上线前零 P0/P1 缺陷，P2 缺陷数 ≤ 5。

### 1.2 测试层级

| 层级 | 范围 | 工具 | 责任人 |
| --- | --- | --- | --- |
| 单元测试 | 工具函数、组件、Store、API handler | Vitest、Jest | 开发 |
| 集成测试 | 模块间联动、API 与数据库 | Vitest + 测试数据库 | 开发 + 测试 |
| E2E 测试 | 用户核心路径端到端 | Playwright | 测试 |
| 功能测试 | 业务功能验证 | 手工 + 用例库 | 测试 |
| 兼容性测试 | 浏览器/设备/分辨率 | 手工 + BrowserStack | 测试 |
| 性能测试 | 响应时间、Lighthouse | Lighthouse、k6 | 测试 |
| 安全测试 | 越权、注入、XSS | 手工 + OWASP 检查清单 | 测试 |
| 回归测试 | 版本迭代与上线前 | 自动化 + 核心用例 | 测试 |

### 1.3 测试方法

- **黑盒测试**为主，覆盖功能用例。
- **白盒测试**辅助，针对核心逻辑（认证、积分事务）审查代码。
- **等价类划分 / 边界值分析**用于表单输入校验。
- **场景法**用于用户流程（注册→验证→登录→查询）。
- **探索性测试**用于补充未覆盖的边界场景。

### 1.4 测试优先级

| 优先级 | 说明 | 必测范围 |
| --- | --- | --- |
| P0 | 阻塞核心流程，必须通过 | 认证、教材查询、校历、新闻列表/详情 |
| P1 | 重要功能，影响体验 | 评论、消息、积分、热门搜索、首页模块 |
| P2 | 边界与异常场景 | 极端输入、并发、兼容性细节 |

### 1.5 测试数据准备

- 测试账号：1 个管理员、5 个普通用户（含已验证 / 未验证 / 已禁用）。
- 业务数据：8 学院 / 20 专业 / 50 班级 / 6 学期 / 5000 教材 / 50 新闻。
- 邮件测试：使用 Mailtrap / Resend 沙箱，避免发送真实邮件。

## 7. 回归测试策略

### 7.1 回归测试时机

- 每次迭代提测时，对已通过功能执行回归。

## 8. 缺陷管理

### 8.1 缺陷等级

| 等级 | 定义 | 处理时效 |
| --- | --- | --- |
| P0 致命 | 核心功能不可用、数据丢失、安全漏洞 | 立即修复，阻塞发布 |
| P1 严重 | 主要功能异常、关键路径受阻 | 24 小时内修复 |
| P2 一般 | 次要功能异常、有 workaround | 本次迭代内修复 |
| P3 轻微 | UI 细节、文案、非功能性建议 | 视情况修复 |

### 8.2 缺陷生命周期

```
新建 → 分派 → 修复中 → 已修复 → 验证中 → 已关闭
                              ↓
                          重新打开
                              ↓
                            修复中
```

- **新建**：测试人员提交缺陷。
- **分派**：负责人分派给开发。
- **修复中**：开发处理。
- **已修复**：开发提交修复，待验证。
- **验证中**：测试回归验证。
- **已关闭**：验证通过。
- **重新打开**：验证未通过，退回开发。

### 8.3 缺陷报告模板

```
标题：[模块] 简要描述
等级：P0/P1/P2/P3
环境：预发 / 生产 / 本地
复现步骤：
  1. ...
  2. ...
预期结果：...
实际结果：...
附件：截图 / 录屏 / 日志
提交人：xxx
提交时间：xxx
```

### 8.4 缺陷统计指标

- 缺陷密度：缺陷数 / 千行代码。
- 缺陷修复率：已修复 / 已发现。
- 缺陷 reopen 率：重新打开 / 已修复。
- 上线遗留缺陷数：按等级统计。
