### 1.3 根因分析

#### 问题1根因：前后台登录状态共享（已在前一轮修复）

**原设计缺陷**：前台 `/login` 和后台 `/admin/login` 共用同一个 localStorage key（`token` + `user`）。

**问题链条**：
1. 管理员通过 `/admin/login` 登录 → token 存入 `localStorage.token`
2. 管理员访问前台点击"登录" → 路由跳转 `/login`
3. 路由守卫检查 `localStorage.token` 存在 → 判断"已登录" → 强制跳转首页
4. 用户看到 `/login` URL 但显示首页内容

**当前状态**：前一轮已修复（独立 `adminToken` / `adminUser` localStorage key），视频录制时部署的是修复前版本。本次部署已确认最新代码生效。

#### 问题2根因：仪表盘无客户端缓存

**原实现**：`DashboardView.vue` 每次 `onMounted` 都调用 `/api/admin/stats` 获取统计数据，服务端虽有 5 分钟缓存，但客户端每次访问都重新请求。

**问题3根因：AdminLayout 无 keep-alive

**原实现**：`AdminLayout.vue` 的 `<router-view>` 未使用 `<keep-alive>`，每次菜单切换都会销毁并重建组件，导致重复 loading。

#### 问题4根因：浏览器/扩展行为

**分析**：代码中"管理员入口"使用标准 `<RouterLink to="/admin/login">`，无 `target="_blank"`。首次点击打开新标签页加载联想默认页面，是浏览器扩展或系统行为，非代码问题。

#### 问题5根因：视频分析误判

**分析**：代码中所有"登录"入口（顶部导航、底部 CTA、教材查询拦截页）均跳转 `/login`，行为一致。视频分析误判为不一致。

## 二、修复方案

### 2.1 修复方案总览

| # | 问题 | 修复方案 | 优先级 | 状态 |
|---|------|----------|--------|------|
| 1 | /login 渲染异常 | 前一轮已修复（独立 adminToken）；本次确认部署最新代码 | P0 | ✅ 已验证 |
| 2 | 仪表盘加载缓慢 | 添加手动刷新按钮，配合 keep-alive 缓存 | P1 | ✅ 已修复 |
| 3 | 后台列表页重复 loading | AdminLayout router-view 添加 keep-alive | P1 | ✅ 已修复 |
| 4 | 管理员入口首次点击异常 | 代码正确，非代码问题（浏览器行为） | P2 | ⚠️ 无需修复 |
| 5 | 登录入口行为不一致 | 代码正确，视频分析误判 | P2 | ⚠️ 无需修复 |

### 2.2 详细修复方案

#### 修复1：AdminLayout 添加 keep-alive（问题2 + 问题3）

**文件**：`src/admin/layouts/AdminLayout.vue`

**修改内容**：
```vue
<!-- 修改前 -->
<main class="flex-1 bg-gray-50 p-4 md:p-6 overflow-auto">
  <router-view />
</main>

<!-- 修改后 -->
<main class="flex-1 bg-gray-50 p-4 md:p-6 overflow-auto">
  <router-view v-slot="{ Component }">
    <keep-alive>
      <component :is="Component" />
    </keep-alive>
  </router-view>
</main>
```

**效果**：
- 后台菜单切换时，已访问的页面从缓存恢复，无需重新 loading
- 首次访问仍会加载，但后续切换即时显示
- 仪表盘统计数据保持，不会因切换菜单而丢失

#### 修复2：DashboardView 添加手动刷新按钮（问题2）

**文件**：`src/admin/views/DashboardView.vue`

**修改内容**：
```vue
<!-- 新增刷新按钮 -->
<button
  type="button"
  class="inline-flex items-center gap-1.5 px-3 py-1.5 text-sm ..."
  :disabled="loading"
  @click="fetchStats"
>
  <svg class="w-4 h-4" :class="{ 'animate-spin': loading }" ...>
    <path ... />
  </svg>
  <span>{{ loading ? '加载中' : '刷新' }}</span>
</button>
```

**效果**：
- 配合 keep-alive 使用，管理员可手动刷新统计数据
- 刷新按钮带 loading 动画，提供清晰的状态反馈

#### 修复3：确认最新代码部署（问题1）

**操作**：
1. 推送 GitHub（3 个提交：5211773、3cbd891、d8cd5ff）
2. 部署 Vercel（部署 ID：dpl_3ABf8VdFeByC2uBrqzXKf1wLPKVb）
3. 配置环境变量（修复 DATABASE_URL 带引号问题）
4. 线上回归测试验证
