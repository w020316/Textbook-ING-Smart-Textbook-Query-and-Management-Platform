## 一、问题分析报告

### 1.1 视频分析概要

对屏幕录制视频 `屏幕录制 2026-07-03 151923.mp4` 全面分析，视频时长约 1 分钟，涵盖：

| 阶段 | 时间段 | 内容 |
|------|--------|------|
| 教材管理 | 00:00 - 00:11 | 列表加载（~4s）、打开新增教材弹窗、关闭 |
| 新闻管理 | 00:11 - 00:13 | 点击新闻管理，未等加载完成即切走 |
| 用户管理 | 00:13 - 00:39 | 改角色（学生→教师→管理员），每次耗时 6-7s，无 loading |
| 校历管理 | 00:39 - 00:46 | 学期列表加载（~2-3s）、打开新增学期弹窗、关闭 |
| 学院管理 | 00:46 - 00:56 | 学院列表加载（~3-4s）、点击"编辑"无响应 |

### 1.2 发现的问题清单

| # | 问题 | 严重程度 | 出现场景 | 复现步骤 |
|---|------|----------|----------|----------|
| 1 | 学院管理"编辑"按钮点击无响应 | **P0** | 学院管理 → 编辑 | 1. 登录管理后台 2. 点击"学院管理" 3. 点击任一学院"编辑" 4. 无弹窗、无响应 |
| 2 | 角色修改无 loading，等待 6-7s | **P1** | 用户管理 → 改角色 | 1. 点击"改角色" 2. 选择新角色 3. 点击"确认" 4. 等待 6-7s 无任何反馈 |
| 3 | 所有写操作缺少成功/失败反馈 | **P1** | 全后台 | 所有创建/编辑/删除操作均无 toast/message 反馈 |
| 4 | 各页面加载慢 2-4s | **P2** | 后台菜单切换 | 已通过 API 缓存优化解决 |

### 1.3 根因分析

#### 问题1根因：学院编辑按钮无响应

**分析**：代码中 `openCollegeEdit(college)` 函数存在且事件绑定正确。可能原因：
- 浏览器缓存了旧版本 JS 文件
- 或点击时 JS 执行出错

**修复策略**：无论根因如何，添加 toast 反馈后，用户能明确感知操作结果。同时确保最新代码已部署。

#### 问题2根因：角色修改无 loading

**原实现**：`submitRole()` 函数直接 `await put(...)`，无 submitting 状态，Modal 确认按钮无 loading 动画，用户不知道是否在处理中。

#### 问题3根因：缺少全局 toast 通知机制

**原实现**：所有后台视图使用 `error.value` 在页面顶部显示错误，用户可能看不到。写操作成功后无任何反馈，用户不知道是否成功。

### 2.1 修复方案总览

| # | 问题 | 修复方案 | 优先级 | 状态 |
|---|------|----------|--------|------|

| 4 | 页面加载慢 | 已在 V3 中通过 API 缓存解决 | P2 | ✅ 已修复 |

### 2.2 详细修复方案

#### 修复1：创建全局 toast 通知系统（问题3）

**新建文件**：
- `src/admin/composables/useToast.ts` - toast composable（单例模式）
- `src/admin/components/ToastContainer.vue` - toast 渲染组件

**特性**：
- 支持 success / error / warning / info 四种类型
- 自动消失（success 3s，error 5s）
- 右上角滑入动画
- 可手动关闭
- 单例模式，所有后台组件共享同一队列

**使用方式**：
```typescript
const toast = useToast()
toast.success('操作成功')
toast.error('操作失败')
toast.warning('请填写必填项')
```

#### 修复2：AdminLayout 引入 ToastContainer（问题3）

**文件**：`src/admin/layouts/AdminLayout.vue`

在布局末尾引入 `<ToastContainer />`，所有后台页面自动获得 toast 能力。

#### 修复3：所有后台视图添加 toast + loading（问题1、2、3）

修改了 5 个后台视图文件：

| 文件 | 修改内容 |
|------|----------|
| `UserManageView.vue` | 角色修改/积分调整：submitting 状态 + loading 动画 + toast 反馈 |
| `CollegeManageView.vue` | 学院/专业/班级 CRUD：submitting + loading + toast + 关闭防护 |
| `TextbookManageView.vue` | 教材 CRUD：submitting + loading + toast |
| `NewsManageView.vue` | 新闻 CRUD + 置顶：submitting + loading + toast |
| `CalendarManageView.vue` | 学期 CRUD + 生成教学周：submitting + loading + toast |

**统一改造模式**：
1. 新增 `const submitting = ref(false)`
2. 写操作前 `submitting.value = true`
3. 成功后 `toast.success('描述')`
4. 失败时 `toast.error(e.message)`
5. finally 中 `submitting.value = false`
6. Modal 关闭函数添加 `if (submitting.value) return` 防护
7. Modal 确认按钮添加 loading spinner 动画 + `:disabled="submitting"`
